## 0. 전제 정리 (이미 파악한 라오어 핵심 — 재조사 불필요)

| 항목 | 내용 |
|---|---|
| 시드 분할 | **40분할** (예: 4,000만 원 → 1회 100만 원 기준) |
| 매수 시점 | **매일 종가 (LOC, Limit-On-Close)** |
| 비중 원리 | 평단가 대비 **하락할수록 더 많이, 상승할수록 적게** 매수 → 평균단가 하락 |
| 전제 시장 | 미국 지수/성장주(TQQQ 등) 장기 우상향 + 변동성 수확 |
| 매도 룰 | "무한매수"는 **매도 없음**(영구 보유). "무한매매"는 익절/리밸런싱 포함 — **용어 구분 중요** (§4) |

## 1. 라오어 매수 로직의 정량 모델 분해

### (a) 매수 트리거
- **이벤트**: 매 영업일, 시장 종가 직전 (LOC 주문 접수 시점).
- **조건**: `remaining_splits > 0` (40분할 잔량 남아있을 때). 잔량 0이면 현금 분할 고갈 → 이후론 "무한매매" 모드(§4)로 전환 또는 분할 재충전(re-arming) 정책 필요.
- **실행**: LOC(종가 제한) 주문 → 슬리피지 최소화, 종가 체결 보장이 핵심. 시장가(MKT) 사용 시 종가 꼬리 체결 불확정성으로 평단가 관리 어려움.

### (b) 비중/수량 결정 함수 (핵심)
입력 → 출력 매핑:

```
입력 변수
  P_now  : 당일 종가(LOC 체결가 추정치 또는 직전 종가)
  P_avg  : 현재 보유 포지션 평균단가 (비용 기준, 수량 가중)
  Q_held : 현재 보유 수량
  Cash_k : k번째 분할 잔액 (남은 현금 / 남은 분할 수)
  k      : 현재 분할 인덱스 (1..40)
  N_left : 남은 분할 수 (40 - (k-1))

출력
  Q_buy  : 당일 매수 수량 (정수, 라운딩)
  Cash_buy = Q_buy × P_now  (실제 현금 지출)
```

**핵심 아이디어**: 편차 `d = (P_now - P_avg) / P_avg` (현재가-평단가 상대편차)를 매수 강도의 입력으로 사용.
- `d < 0` (현재가 < 평단가, 즉 하락) → **더 많이 매수**
- `d > 0` (현재가 > 평단가, 즉 상승) → **적게 매수**
- 이는 평균단가를 *능동적으로* 끌어내리는 방향의 자본 배분.

### (c) 40분할 스케줄링
- **기준 예산**: `Base_k = TotalSeed / 40` (예: 100만 원/일). 단순 DCA라면 매일 `Base_k` 고정.
- 라오어 변형: `Base_k`를 편차 `d`로 변조 → 하락일은 `> Base_k`, 상승일은 `< Base_k`.
- **원금 고갈 조건**: 40 영업일(≈ 2개월) 경과 후 잔량 0. 이후:
  - 옵션 A: 전략 종료(순보유 전환)
  - 옵션 B: "무한매매" 모드 전환 — 이미 보유 분량의 일부 익절 → 현금 재확보 → 분할 재충전(re-arm). (라오어 책에서 다루는 "무한매매"는 대체로 이 쪽)
  - 옵션 C: 신규 자본 주입 (외부 시드 추가)
- **조정(리밸런싱) 조건**:
  - 평단가 대비 현재가가 일정 역치(+x%) 초과 상승 시 → 익절(무한매매의 경우) 또는 매수 중단(무한매수의 경우).
  - 극단 하락(-y%) 시 → 분할 예산 전량 소진 위험 → `Cash_buy` 상한 캡핑 또는 분할 재배분.

### (d) 매도 룰 — "무한매수" vs "무한매매" (차이 명시)

| 전략 | 매도 | 비고 |
|---|---|---|
| **무한매수** | **없음** | 영구 보유. 분할 매수로 평단가만 관리. 장기 우상향 전제에 베팅. |
| **무한매매** | **있음** | 평단가 대비 +목표수익률 도달 시 부분 익절 → 현금화 → 재분할 매수 사이클. 일종의 **변동성 수확(volatility harvest)** + 드리프트 정규화. |

라오어는 보통 후자(무한매매)를 더 넓은 의미로 사용하나, "무한매수법"이라 명명할 땐 전자(매도 없음)에 가까운 기본형을 가리키는 경우가 많음. **세미나에서는 양쪽을 분리 제시 권장.** (추측: 라오어 원서의 정확 용어 사용은 확인 필요)

## 2. 비중 함수 후보 수식 (2개 제안)

공통 기호:
- `d = (P_now - P_avg)/P_avg` (상대편차, 하락이면 음수)
- `Base = TotalSeed/40` (1회 기준 예산)
- `λ ≥ 0`: 편차 민감도(하이퍼파라미터, 백테스트로 튜닝)
- `Q_buy`, `Cash_buy` 는 라운딩 후 상한(남은 분할 예산)으로 캡핑.

### 후보 A — 선형 반비례 (Linear Anti-Drift)

```
Cash_buy = Base × (1 - λ·d)
Q_buy    = Cash_buy / P_now
```

- **입력**: `Base, d, λ, P_now`
- **출력**: 매수 현금액(→ 수량)
- **거동**:
  - `d=0` → `Base` 그대로 (순수 DCA와 동일)
  - `d<0`(하락, 예 -10%) → `Base×(1+0.1λ)` → 더 많이 매수
  - `d>0`(상승) → `Base×(1-λ·d)` → 적게 매수, `d≥1/λ`이면 매수 0 또는 매도(음수 → 캡핑)
- **장점**: 단순·해석 가능·백테스트 파라미터 1개. **단점**: 극단 하락에서 비용이 선형 폭증 → 캡핑 필수.

### 후보 B — 계단식/시그모이드 (Step / Sigmoid Anti-Drift)

```
Cash_buy = Base × clamp( 1 / (1 + e^(k·d·s)), q_min, q_max )
```
또는 단순 계단:
```
d < -8%  : 2.0×Base
d ∈ [-8%,-3%] : 1.5×Base
d ∈ [-3%,+3%] : 1.0×Base
d ∈ [+3%,+8%] : 0.5×Base
d > +8%  : 0 (또는 익절, 무한매매 시)
```

- **입력**: `Base, d` + 구간 파라미터(계단) 또는 `(k,s,q_min,q_max)`(시그모이드)
- **출력**: 매수 현금액(→ 수량)
- **장점**: 극단 구간에서 비용이 상한 처리됨 → 원금 고갈 리스크 통제 우수. 라오어식 "직관적 비중 조절"과 부합. **단점**: 파라미터 다수 → 과적합 위험, 백테스트로 정규화 필요.

### 결론 (추천)
- **v1 프로토타입은 후보 A(선형)**로 빠르게 검증 → 파라미터 `λ` 1개로 동작 특성 파악.
- **운용/세미나 시연은 후보 B(계단)** 채택 → 리스크 통제·직관성·설명력 우수. 라오어 원서의 분위기와도 부합.
- 양쪽 모두 **캡핑(upper bound)과 최소 주문 단위 처리**는 필수.

## 3. 학술/정량 근거 (각 한 줄)

| 개념 | 한 줄 근거 + 출처 |
|---|---|
| **DCA (비용평균법)** | 고정 금액 정기 매수 → 가격 하락 시 더 많은 수량 매수, 자연스럽게 평단가 < 평균가 형성. 가장 검증된 공식 투자법. ([Bogleheads: DCA](https://www.bogleheads.org/wiki/Dollar_cost_averaging)) |
| **Value Averaging (Edleson)** | 목표 포트폴리오 가치 곡선 설정 → 편차만큼 매수/매도 조정. DCA보다 **더 낮은 평단가·더 높은 수익률**로 보고됨 (Edleson 테스트: VA 17.03% vs DCA 15.74%). 라오어 비중 함수의 가장 가까운 학술 대응물. ([Edleson, *Value Averaging*](http://www.valueaveraging.ca/); [J. Financial & Strategic Decisions 비교 연구](https://www.bogleheads.org/wiki/Value_averaging)) |
| **평균단가 하락 효과** | 변동성 있는 자산에서 "하락 시 더 사면" 기하평균 < 산술평균 효과로 평단가가 단순 시간평균가보다 낮아짐. 단, **트렌드 하락장에서는 효과 상쇄/역전** 가능 (전제: 장기 우상향 필수). |
| **마틴게일 / anti-martingale** | 라오어의 "하락 시 추가 매수"는 **anti-martingale(승자 강화)**이 아닌 **마틴게일 계열(손실 시 베팅 증가)**의 자본 배분 변형. 단, 주식 DCA 맥락에서는 무한 자본·무한 시간 가정이 성립하므로 카지노 마틴게일의 파산 위험과는 다름. 자본 한도(40분할)가 파산 방지장치. |
| **Time diversification** | 매입 시점 분산 → 특정 시점 집중 리스크(타이밍 리스크) 축소. 다만 학술적으로는 주식 기대수익률이 시간에 대해 거의 확정적이므로 ** lump-sum이 기대값상 우위**라는 반론도 있음 (DCA는 분산·심리 효과 차원에서 정당화). ([Bogleheads: Lump sum vs DCA](https://www.bogleheads.org/wiki/Dollar_cost_averaging#Lump_sum_versus_DCA)) |
| **변동성 수확 (volatility harvest / drift)** | 변동하는 자산에서 정기 리밸런싱/재투자 시 **드리프트**를 이용해 저가 매수·고가 매도 효과 발생 → 장기 초과수익. 무한매매(익절+재매수) 모델의 이론적 기반. ([Rebalance bonus / Shannon's demon](https://www.bogleheads.org/wiki/Rebalancing)) |

**총평**: 라오어 무한매수법은 **"DCA + Value Averaging의 하이브리드 + 자본 한도(40분할) 마틴게일 안전장치"**로 정합적으로 해석 가능. 완전히 새로운 이론이 아니라 **기존 정량 투자 이론의 실용 조합**.

## 4. 자동매매 구현 명세 (개념적 — 코드 없음)

### 4.1 입력 데이터 (Data Inputs)
| 데이터 | 소스(개념) | 용도 |
|---|---|---|
| 당일 종가 예측/직전 종가 | 브로커行情 | LOC 주문 가격 |
| 평단가(P_avg) | 브로커 잔고 API | 편차 `d` 계산 |
| 보유 수량(Q_held) | 브로커 잔고 API | 평단가 갱신·리스크 한도 |
| 남은 분할 수(N_left) | 내부 상태 | 스케줄 종료·캡핑 |
| 잔여 현금(Cash_remaining) | 브로커 잔고 API | 매수 예산 상한 |

### 4.2 출력 (주문)
- **주문 타입**: LOC (Limit-On-Close) 또는 브로커 지원 종가 주문 유형
- **주문 내용**: `{ ticker, side=BUY, quantity=Q_buy, type=LOC, price=당일종가추정치 }`
- 라운딩: 소수점 수량 불가 시 정수 처리, 잔금 누적(reinvest) 정책

### 4.3 상태 (State Machine)
```
state = {
  total_seed, n_splits=40, k(현재분할),
  cash_remaining,            # 남은 매수 예산
  position: { qty, avg_cost },  # 보유 포지션
  params: { λ 또는 계단 구간 },  # 비중 함수 파라미터
  mode: BUY_ONLY | BUY_SELL,    # 무한매수 vs 무한매매
  thresholds: { dip_cap, sell_target }  # 캡핑/익절 역치
}
```

### 4.4 실행 주기 (Schedule)
- **트리거**: 매 영업일, 미국 시장 종가 약 5~10분 전 (LOC 주문 접수 마감 전)
- **타임존**: ET (한국 시간 새벽 3:30~4:00 전후, 서머타임 변동)
- **주기 핸들**: cron/스케줄러 1회/일 + 시장 휴장일 필터(Holiday calendar)
- **재시도/예외**: 체결 미확인, 부분 체결, 브로커 API 장애 시 다음 영업일 이월 규칙 정의 필요

### 4.5 난점 (구현 리스크)
1. **LOC 주문 미지원 브로커** → 대체(MOC, 또는 종가 직전 시장가) 시 슬리피지 → 평단가 관리 정확도 저하.
2. **평단가 계산 기준**: 세금/수수료/배당 포함 여부(비용기반 vs 단순가격기반)에 따라 `d`가 달라짐 → 정의 표준화 필요.
3. **40분할 고갈 후 정책 미정** → 무한매매 전환 로직(익절 역치, 재분할 크기) 별도 설계 필요.
4. **극단 하락(예: -30%)**에서 비용 폭증 → 캡핑 없으면 분할 예산 조기 소진.
5. **데이터/상태 동기화**: 브로커 잔고와 내부 상태 불일치 시 편차 오계산 → 정합성 검증(리컨실리에이션) 로직 필요.
6. **시장 휴장·반일장** 캘린더 관리.

## 5. 정량 분석 담당 평가 (한 줄)

라오어 무한매수법은 **기존 정량 이론(DCA+Value Averaging)의 실용 조합으로 정합적**이며, 비중 함수(후보 A→B)와 자본 한도 안전장치만 잘 정의하면 **알고리즘화 충분히 가능**하지만, 성패는 "장기 우상향 전제(TQQQ 등)"와 "LOC 주문 체결 정확도·평단가 정합성"이라는 **실행 인프라**에 달려 있음 — 전략 자체의 우아함보다 구현 충실도가 수익을 좌우.
