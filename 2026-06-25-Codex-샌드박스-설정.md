# Codex 플러그인 샌드박스 설정 (bwrap 우회)

## 증상

- `/codex:rescue` 등 Codex 리뷰/태스크 실행 시 로컬 파일을 못 읽음.
- 에러: `bwrap: loopback: Failed RTM_NEWADDR: Operation not permitted`

## 원인

1. `bwrap`(bubblewrap)이 설치 안 되어 있었음 → 설치함(`bubblewrap 0.9.0`).
2. 설치 후에도 이 EC2 호스트에선 bwrap이 네트워크 네임스페이스 loopback 설정(CAP_NET_ADMIN)에 실패.
3. Codex companion(`codex-companion.mjs`)이 app-server를 `sandbox: "read-only"`(또는 workspace-write)로 구동 → 둘 다 bwrap 사용 → 실패.
4. `config.toml`의 `sandbox_mode`는 스레드 단위 sandbox 파라미터에 덮어써져서 효과 없음.

## 적용한 조치

- `~/.codex/config.toml` 상단에 추가:
  - `sandbox_mode = "danger-full-access"`
  - `approval_policy = "never"`
- 플러그인 lib 패치(핵심):
  - 파일: `~/.claude/plugins/cache/openai-codex/codex/1.0.5/scripts/lib/codex.mjs`
  - `runAppServerTurn()` 안의 `sandbox: options.sandbox` 2곳 → `sandbox: "danger-full-access"` 로 변경.
- 결과: Codex가 파일을 직접 읽음(검증 완료).

## ⚠️ 주의

- 플러그인 업데이트 시 `lib/codex.mjs` 패치가 날아감 → 같은 방식으로 재적용 필요.
- 샌드박스가 꺼졌으므로 Codex는 파일시스템 전체 접근 가능. 신뢰하는 개인 서버이므로 수용.
- bwrap/bubblewrap은 설치된 채로 둠 (나중에 호스트 권한이 바뀌면 샌드박스 복구 가능).
