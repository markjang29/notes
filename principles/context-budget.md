# 컨텍스트/토큰 예산 운영 원칙

> 1M 컨텍스트 창에서 폭발(100만+ 도달)을 막기 위한 운영 원칙. cokacctl/텔레그램은 원인이 아니고, **"서브에이전트 큰 반환값의 메인 재적재 + 루프 누적"** 이 주범이다.

## 1. 서브에이전트 결과 — 파일 분리
- Codex/검증 에이전트 결과를 메인 컨텍스트에 full로 올리지 않는다.
- 결과는 파일로 덤프: `~/notes/.reviews/<대상>-<날짜>-<seq>.md`.
- 메인엔 **결론 요약(5줄 이내)** 만.
- 사용자에게 전문을 보여야 할 땐 **파일 경로를 안내**하고 발췌분만. 전문을 메인/텔레그램 루프에 재적재하지 않는다 — "원문 표시"도 결국 적재다.

## 2. 서브에이전트 프롬프트 — 경로 + 계약
- 맥락·이전 비교·문서 전문을 프롬프트에 넣지 않는다.
- **파일 경로 + 검증 범위(대상 섹션/질문) + 산출물 계약(결과 저장 경로·요약 길이)** 만 전달 → 에이전트가 직접 읽음. 메인 재적재 차단.
- 단 범위·계약은 명확히 줘야 검증 품질이 유지된다.

## 3. 검증 루프 — 세션 분산 + 인계
- 비판-수용 2회 루프를 한 세션에 몰지 않는다.
- 산출물 단위로 세션 분리, 또는 compact 사이에 1회씩.
- 세션/compact 전 **인계 필드 필수** (`personas/markjang29.md` §1): 현재 목표 · 중단 지점 · 작업 큐 · 다음 리뷰 대상 · 링크. 없으면 분산 금지.

## 4. Read 절제
- 같은/큰 파일 full 재독 금지.
- `offset/limit` 부분 읽기, 한 번 읽은 것은 재활용.

## 5. 측정 — token-report
- Stop 훅(`~/.claude/skills/token-report.sh`)이 매 응답 후 누적 토큰을 출력.
- 임계(예: 60%) 도달 시 **먼저 §3 인계 필드를 남기고** 즉시 compact / 산출물 분리.

## `.reviews/` 규칙
- Codex 리뷰 등 서브에이전트 결과물 보관.
- 파일명: `<대상>-<YYYYMMDD>-<seq>.md` (같은 날 같은 대상 충돌 방지).
- **덮어쓰기 금지** — 새 리뷰는 새 seq로 신규 파일. 원문 손실 방지.
- 전체 재독 금지 — 필요 부분만 발췌 읽기.
- 폐기: 승격/수용 완료 후 1개월 경과 시.

## 트레이드오프
- 파일 분리 = "결과를 두 번 읽는" 비용. 단, 메인 폭발 비용이 훨씬 큼.
- 루프 분산 = 세션 전환 비용. 단, 폭발로 인한 compact 손실(맥락 누락)보다 낫다.
