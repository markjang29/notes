# 2026-07-02 — rpg_game: 시그니처 확정 (참모 추론 × 지휘관 결정 × 패막 손맛)

> 참여: 이사님(한 준희) + RPG 팀장 에이전트(Linux 서버) · 기획 아이디에이션 → Codex 검증 → 확정

## 배경
- 직전까지 시그니처 후보 탐색 중 (CIPHER 해독 / LUMEN 가시선 / RUMOR 소문 — `IDEATION.md`).
- 이사님이 본 세션에서 독자 시그니처 방향을 제시 → 팀장 정리 → Codex 검증 → 이사님 최종 확정.

## 흐름

1. **이사님 아이디어 제시** — 전투 캐릭터가 "전투를 어떻게 이길지 생각하는 과정"을 **AI Claude가 사고하듯 시각화**. 레벨업 = 추론의 정교화(숫자 스탯이 아니라 "사고의 질"로 성장). 글로 화면에 뜸. "이 루트 기대 데미지 최대 → 시도" 같은 추론 언어를 사용자가 쾌감으로 체험.
2. **이사님 확장** — 캐릭터는 전투 추론뿐 아니라 **메타 루프(다음 아이템/던전/재료)까지 추천**. "토큰" 재화로 추론에 비용. 핵심: **AI가 잼병인 건 옷 색깔** → 걷기로 모은 포인트로 룩을 맞춘다(지능축=레벨 / 룩축=걷기 분리).
3. **Codex 검증** (`codex:rescue`, 새 스레드) — 4각도(독창성·구조 모순·모바일 적합·프로토타입) 진단. 결론: **"수정 채택 권고"**. 리스크 2 — ① 추론 품질이 전투력으로 굳음(공정성 붕괴) ② 재미가 전술이 아니라 문장 읽기가 됨(오토배틀러화). 채택 조건 3 + 프로토타입 2종 제안.
4. **이사님 결정적 콜** — "오토배틀러화 방지 장치 = **패링**". Codex의 핵심 리스크(②)를 이미 검증된 첫 모듈(parry)로 해결. `grid_parry` 위에 `reasoning`이 얹히는 구도로 자산이 낭비 없이 통합.
5. **시그니처 확정** — "참모(추론) × 지휘관(결정) × 손(패링)" 삼각 구도. 다음 모듈 **B안**(reasoning × grid_parry 통합 프로토타입) 콜.

## 결정

1. **시그니처 확정** — 참모 추론 × 지휘관 결정 × 패링 손맛 (`08-reasoning-parry-signature.md`).
   - 캐릭터 = "생각은 천재, 패션은 잼병인 AI" (지능=레벨 축 / 룩=걷기 축, 분리).
   - 걷기→전투력 0기여(`06` 캡③) 유지 — 전투력(추론)은 걷기와 단절.
2. **Codex 채택 조건 3 수용**:
   - ① PvP 추론/토큰/정보 완전 정규화 (랭크 양측 동일 경기용 두뇌).
   - ② 캐릭터 성장 추론은 PvE·스토리 중심 (PvP는 평탄).
   - ③ 보고서 = 짧은 칩 UI (한 줄 결론 + 근거 칩 ≤3 + 접기).
3. **참모 보고서 원칙** — "정답 제시" ❌ / "편향 있는 2~3개 작전안, 지휘관이 교정" ✅. 캐릭터별 편향·시야 좁음·정답률 100% 금지. 빈틈을 패링(손)이 메움.
4. **다음 모듈 B안** — `reasoning` × `grid_parry` 통합 프로토타입. (이사님 콜)
