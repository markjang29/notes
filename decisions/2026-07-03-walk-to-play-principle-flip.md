# ADR — Walk-to-Play 원칙 전복 (걷기→전투 윈도우 직결, 사고 NO)

## 상태
accepted (2026-07-03, 이사님 세션 확정) — 선행 원칙 **"걷기→전투력 0기여"**(`06 §4 캡③`, `08 §3·§4`)를 **전복**.

## 날짜
2026-07-03

## 프로젝트 / 적용 범위
rpg_game — 걷기×전투 연결 원칙 전반. PvE/PvP 분리 구조.

## 결정
1. **"걷기→전투력 0기여" 원칙 폐기** → **Walk-to-Play**: 걷기는 전투 **윈도우(시간/인내)** 로 직결.
2. 걷기 = 보편 재화. 소비처 **분기**: **룩(표현) 또는 전투 윈도우(+초)** — 플레이어 선택.
3. **사고(AI 추론 품질)에는 연결 NO.** 시간(윈도우)만. (정체성 "생각 천재" 보존)
4. **PvE(캠페인)** = Walk-to-Play 직결 / **PvP(진검승부 듀얼)** = 분리, 동일 표준 윈도우로 정규화(걷기는 PvP에선 룩으로만).
5. **캡**으로 환산 상한(~7000보 ≈ +0.7초) → 치팅 무의미화 + 건강 상한.
6. 메커니즘 = **심리적 우위**(간보기·버티기). 더 늦게까지 버틸 수 있는 쪽이 우위, **패링**이 결정적 역전.

## 맥락
- 2026-07-03 이사님 세션: 크래시 게임 cash-out 비유 → "걷기→공격 윈도우" 제안. 사무라이 진검승부 예시(기본 5.0초 → 7000보 = 5.7초).
- 기존 "0기여" 원칙은 P2W 우려에서 비롯. 그러나 이사님 프레임: **"돈 내는 놈이 강한 건 양심 없지만, 걸은 놈이 강한 건 정당하다"** — 과금 P2W가 아닌 **행동=전투력**(Walk-to-Play) 철학.
- 진검승부 본질 = "누가 더 늦게까지 버티느냐"(간보기). 걷기가 그 인내 시간을 준다는 통찰 — 단순 수치 업이 아니라 심리전 우위로 번역.

## 제약
- 모바일(세로, 한 손 엄지), Telegram 비대화형.
- 치팅(가짜 보행/센서 조작) 위험 — 센서 정교화 + 캡으로 완화.
- 기존 정체성 "생각 천재·패션 잼병"(지능=레벨 / 룩=걷기 분리, `08 §3`)과의 정합성 흔들림 리스크.

## 선택한 이유 / 버린 대안
- **선택: 시간만, 사고 NO**
  - 정체성(지능=레벨 축) 보존 — 지능이 걷기에 묶이지 않음.
  - 시간=행동경제, 지능=스탯 — 성격이 달라 분리가 자연스러움.
- 버린 **걷기→사고(추론 품질) 연결** — `08 §4` 폐기안(추론품질↔걷기 = 우회 T2W) 회귀 + 정체성 붕괴. (후속 재검토 여지는 둠)
- **선택: PvE/PvP 분리**
  - 정합성 충돌 제로. PvE는 Walk-to-Play, PvP는 순수 심리전+패링.
- 버린 **PvP도 실제 걷기 그대로** — Walk-to-Win 심화 + 보행 환경 차별 + 치팅 최대화.
- 버린 **PvP만 정규화·PvE 모노리스** — 모드/게임 단위 분리가 정합성·난이도 조율에서 더 깔끔.
- **선택: 캡** — 치팅 무의미 + 건강 상한(과도한 보행은 오히려 건강 악영향).
- 버린 **무제한 비례** — 치팅 유인 + 건강 악영향 + 밸런스 붕괴.

## 트레이드오프
- **얻은 것**: Walk-to-Play 차별점(과금 아닌 행동=전투력) / 걷기 가치 다층화(룩+윈도우) / 심리전 깊이 / 치팅 구조적 무의미화 / "싸우기 싫은 사람" 룩 경로 분리.
- **잃은 것**: "걷기→전투력 0기여"라는 단순 불변원칙 / 보행 환경 차별 논란 잔존(완화만, 제거 아님) / 정체성-룩축 정합성 미세 흔들.
- **룩축 재정의**: 걷기는 "룩 + 윈도우" 둘 다 주지만, **룩 자체는 전투력이 아님**(코스튬 스탯 無). 구조는 유지, "0기여"만 폐기.

## 검증 기준
- [ ] 사무라이 진검승부 프로토타입(`demo/modules/duel_advance/`) 폰 검증 — 심리전(간보기/버티기) + 패링 역전 체감.
- [ ] 캡 튜닝값(~7000보/+0.7초) 적정성 — 과하지 않은지.
- [ ] PvE/PvP 분리가 정합성 충돌 없이 동작하는지 확인.
- [ ] **Codex 재검증 필수** — 원칙 전복(대원칙 변경)이므로 자체 점검만으로는 Go 불가(`change-deployment-checklist` 12단계 + Codex 검증).

## 실패 / 복구 과정
- 프로토타입 전 단계. 실패 기록은 폰 검증 후 추가 예정.
- **주의함정**: "걷기→사고 연결"로 다시 빠지지 않도록(`08 §4` 폐기안 회귀 감시).

## 후속 재검토 조건
- PvE 캠페인 검증 후, 사고(추론) 연결 여부 재판단 가능(현재 NO).
- 캡 수치 튜닝은 본편 조립 시점(`DEVELOPMENT.md` 튜닝 철학 준용).
- PvP를 "모드"로 할지 "별도 게임(별품)"으로 할지 — 프로토타입 검증 후 판단.
