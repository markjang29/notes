# decisions/

주요 **결정과 트레이드오프 기록** (ADR — Architecture Decision Record). 의도부채 방어용.

**프롬프트에만 남은 의도는 사라진 것으로 본다.** 한 결정마다 한 파일. 템플릿: `ADR-template.md`.

## ADR 작성 트리거 (이런 변경엔 필수)
- 스택·언어·엔진·주요 라이브러리 선택
- 되돌리기 어려운 구조·아키텍처 변경
- 외부 서비스·API 채택·교체
- 성능·보안·비용 트레이드오프를 수반한 결정
- "왜 이렇게 했지?"를 나중에 물을 수 있는 결정

## 양식 (요약)
결정 내용 / 선택 이유 / 버린 대안 / 당시 제약 / 트레이드오프 / 검증 기준 / 실패·복구 / 후속 재검토 조건 (→ `ADR-template.md` 전체 필드)
