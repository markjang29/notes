### 대조에서 발견 — 5종 밖이지만 엔진에 영향 주는 **추가 요구사항** (보강 제안)
> 매니저 지정 5종엔 없으나 수렴안에 있어 엔진/SDK 요구로 직결되는 항목들. 비교표 작성 시 포함 제안.

- **a. 인앱결제 + 보상형 광고 SDK** — `06 §10` 수익 모델의 실행 인프라. 엔진별 SDK 성숙도 차이 큼.
- **b. 백그라운드 보행 센서 + 배터리** — 걷기가 입장재화(`06 §3`)라 포그라운드/백그라운드 센서 동작 + 배터리 효율이 핵심. 네이티브 권한·OS 제약 직결.
- **c. 크로스 진영 매칭 + 소수 진영 보너스**(`06 §8`) — 콘텐츠보단 서버 로직이나, 멀티플랫폼 계정/매치메이킹 일관성에 영향.
- **d. 캐릭터 로맨스·코스튬 표현**(`06 §7/§10`) — 2D 스프라이트/애니메이션 파이프라인, 의상 교체 렌더링 부담.

## 제약
- **인디 예산/인력** — 상용 엔진 수수료·런타임 비용·러닝커브가 직격. 로열티 0 또는 저비용 우대.
- **모바일 1순위(iOS/Android)** — APK/AAB·App Store 심사·백그라운드 센서 정책(특히 iOS 건강/위치 권한) 준수.
- **P2W/T2W 차단 원칙**(`06 §2/§4`)은 엔진과 무관하나, **클라이언트 신뢰(조작/치트 방지)** 측면에서 서버 권위 설계 필요 → 엔진의 네트워킹/직렬화 능력 간접 관련.
- **커뮤니티/생태계** — 인디 팀 = 튜토리얼·에셋·포럼 접근성이 생산성 직결.
