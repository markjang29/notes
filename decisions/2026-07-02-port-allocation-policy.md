# ADR — 포트 할당 정책

## 배경
이사님 07-02 지시: "전체적으로 포트라던지 구분 잘해서 충돌 안 나게 관리해. 나중에도 웹 여러 개 띄울 것 같으니까."
서버 현황(07-02): 열린 포트 = 22(SSH)·53(DNS)·8788(node). 거의 전부 여유. Docker·Oracle 미설치.

## 포트 대역 할당 (확정)

| 대역 | 용도 | 비고 |
|---|---|---|
| **8000-8099** | 내부 REST API 서비스 (FastAPI 등) | autotrader 백테스트 API = **8001** |
| **8500-8599** | 대시보드/프론트 (Streamlit 등, 옵션) | API와 분리 |
| **1521** | Oracle DB (표준) | Oracle Free 23ai |
| **80 / 443** | nginx 리버스 프록시 (외부 노출 유일) | 도메인/경로별 라우팅 |
| 22 | SSH | 기존 |
| 8788 | 기존 node 서비스 | 건드리지 않음 |

## 원칙
1. **외부 직접 노출 금지** — 모든 서비스는 nginx(80/443) 뒤에서 경로/도메인 라우팅. 서비스 포트 자체는 127.0.0.1 바인딩.
2. **서비스 등록 시 ports.md(본 문서)에 즉시 기록** — 포트 번호·서비스명·소유 팀장·시작일.
3. **충돌 전 사전 확인** — `ss -tlnp` 로 점유 확인 후 할당.
4. 신규 웹은 8000번대 순차 할당(8002, 8003 ...).

## 매핑 (현재)
- 8001 → autotrader 백테스트 REST API (trader 팀장)
- 1521 → Oracle DB Free 23ai (매니저 세팅)

## 차기
- nginx 설치 + 리버스 프록시 기본 라우팅 세팅 (매니저 ops).
- 도메인 확정 시 80/443 가상호스트 추가.
