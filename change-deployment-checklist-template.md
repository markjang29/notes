# 변경 건 도입 전 체크리스트 (일반 템플릿)

> 사칙(`~/notes/onboarding.md` § 변경 건 처리 방법론)에서 의무화한 12단계 검증 템플릿.
> 사용법: 이 파일을 `~/notes/{변경건}-checklist.md`로 복사 후 채운다. (예: `healthcheck-deployment-checklist.md`)

## 1. 구조 검증
- [ ] 필수 파일 존재 (스킬: SKILL.md / 스크립트: 본체)
- [ ] 메타데이터·구성 정상 (스킬: YAML name/description / 스크립트: shebang·권한)
- [ ] 외부 참조(리소스·import·경로) 정상 연결

## 2. 단위 테스트
- [ ] 컴포넌트별 개별 동작 확인
- [ ] 정상 입력 → 정상 출력
- [ ] 비정상 입력 → 정의된 에러 처리
- [ ] 로그·시간대·인코딩 정상

## 3. 통합 테스트
- [ ] 엔드투엔드 동작
- [ ] 실패 시나리오 (에러코드/장애 주입)
- [ ] 폴백·복구 경로 확인
- [ ] 스킬인 경우: Claude가 자동/명시적 호출로 인식·실행

## 4. 온보딩 테스트 (각 봇)
- [ ] 매니저 인지·통합
- [ ] 팀장(RPG/Trader/Scenario) 인지·통합
- [ ] 온보딩 인증(핵심 1줄 인용) 통과

## 5. 운영 환경 검증
- [ ] 환경 변수 확인 (필수 키, GLM 프록시는 AUTH_TOKEN/BASE_URL/DEFAULT_*_MODEL)
- [ ] 디스크/리소스 여유
- [ ] 권한 (실행비트, 소유자)

## 6. 스케줄러 검증 (cron/예약 사용 시)
- [ ] dry-run 1회 실행
- [ ] 로그 정상 기록
- [ ] 실제 cron 등록 후 첫 주기 확인

## 7. 장애 복구 훈련
- [ ] 주요 장애 시나리오 모의 (예: 429/529/quota/auth)
- [ ] 각 시나리오별 감지→대응→복구 확인
- [ ] emergency/보존 산출물 정상 생성

## 8. 문서 완성도
- [ ] 원칙/방법론 문서 존재
- [ ] 사용 가이드(빠른 참조) 존재
- [ ] `onboarding.md`에 의무 읽기/통합 조건 반영

## 9. 성공 기준
- [ ] 기능적 요구사항 달성
- [ ] 비기능(성능: 실행시간/크기) 기준 충족
- [ ] 신뢰성(거짓 양성/음성 비율) 기준 충족

## 10. 롤백 계획
- [ ] 롤백 트리거 명시
- [ ] 롤백 절차 단계화
- [ ] 롤백 후 기존 상태 복원 방법

## 11. 도입 결정 (Go/No-Go)
- [ ] Go 조건 모두 충족 (위 항목 전부 통과)
- [ ] No-Go 조건(치명 실패) 없음
- [ ] Codex/동료 재검증 판정: {GO / GO with conditions / NO-GO}
- [ ] 최종 판정: {Go / No-Go} (이사님/담당 승인)

## 12. 도입 후 모니터링
- [ ] 첫 24시간: 에러/성능/로그 추적
- [ ] 첫 1주일: 일일 리포트·이상 즉시 대응
- [ ] 피드백 수집 후 개선 반영

- **Phase 1:** 기본 검증 (1, 2)
- **Phase 2:** 통합 테스트 + 운영 환경 (3, 5)
- **Phase 3:** 온보딩 (4)
- **Phase 4:** 운영 투입 준비 (6, 7, 8)
- **Phase 5:** 도입 결정 (9, 10, 11) → 통과 시 12 진입
